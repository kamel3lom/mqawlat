# ملاحظات تطوير علومكم للمقاولات

هذا المجلد مخصص للتوثيق الداخلي للمشروع `kamel3lom`.

## مبادئ النسخة الحالية

- التطبيق Static بالكامل ويعمل على GitHub Pages.
- لا يوجد Backend.
- لا توجد مفاتيح API داخل الواجهة.
- كل البيانات القابلة للتعديل موجودة داخل `data`.
- بعض أسعار السعودية مربوطة بمصدر GASTAT الرسمي، أما بقية البنود أو الدول الموسومة `isDemo: true` فهي تجريبية ويجب استبدالها قبل الاستخدام العملي.

## إضافة دولة جديدة

1. أضف الدولة إلى `data/countries.json`.
2. أضف أسعار المواد إلى `data/prices.json`.
3. أضف أجور العمالة إلى `data/labor-rates.json`.
4. أضف كود البناء المرجعي إلى `data/building-codes.json`.
5. أضف معامل الدولة في `data/calculation-factors.json` إذا لزم.

## إضافة مصدر رسمي للأسعار

1. أضف المصدر إلى `data/official-price-sources.json`.
2. لا تجعل `isDemo` مساوية لـ `false` في `prices.json` إلا إذا كان السعر مأخوذًا من مصدر رسمي أو عرض موثق.
3. إذا كان المصدر PDF أو Excel دوريًا، أضف Parser محافظ داخل `scripts`.
4. يجب أن يفشل السكربت عند عدم العثور على الصفوف المتوقعة بدل إدخال أسعار افتراضية.

## إضافة مادة جديدة

1. أضف المادة إلى `data/materials.json`.
2. أضف معامل الكمية داخل `materialQuantityFactors` في `data/calculation-factors.json`.
3. أضف أسعار المادة لكل دولة داخل `data/prices.json`.

## قوالب المخططات التخيلية

ملف `data/design-templates.json` يتم توليده من `scripts/generate_design_templates.py`.

- لا تعدل الملف الكبير يدويًا إلا للضرورة.
- عدل الطرازات أو أنواع التصميم داخل السكربت ثم أعد توليد الملف.
- القوالب تخيلية فقط ولا يجب تقديمها كمخططات هندسية معتمدة.

## مراجعة الحسابات

الأرقام والمعاملات في `calculation-factors.json` تقديرية. عند تحويل المشروع إلى أداة إنتاجية، يجب مراجعتها بواسطة مهندس كميات أو مكتب هندسي.
