**وظيفة الأداة:**  
تعمل الأداة على مساعدة أصحاب المشاريع، والطلاب، والباحثين، والمهتمين بمجال المقاولات والبناء في تكوين تصور أولي سريع عن تكلفة المشروع وكمياته الأساسية قبل الانتقال إلى مرحلة التسعير التفصيلي أو مراجعة المكتب الهندسي. الأداة تعرض تقديرات للمواد، والعمالة، ومراحل التنفيذ، ومصادر الأسعار، مع إمكانية تصدير تقرير قابل للطباعة يحمل بصمة **kamel3lom**. ويجب التنبيه بوضوح إلى أن نتائج الأداة تقديرية وإرشادية فقط، وليست عرض سعر رسميًا، ولا حصر كميات معتمدًا، ولا بديلًا عن المهندس المختص أو المكتب الهندسي أو المقاول المرخص.

**طريقة استخدامها للمبتدئين:**  
افتح رابط الأداة من المتصفح، ثم اختر الدولة والمدينة أو المنطقة، وحدد نوع المشروع، وبعد ذلك أدخل مساحة الأرض ومساحة البناء وعدد الطوابق. إذا كان المشروع يحتوي على قبو، أو سور خارجي، أو خزان أرضي، أو خزان علوي، أو مصعد، فقم بتفعيل الخيارات المناسبة. بعد ذلك اختر نوع البناء، سواء كان عظمًا بدون تشطيب أو بناءً مع التشطيب الكامل، ثم اضغط على زر **احسب الآن**. ستظهر لك لوحة النتائج وفيها التكلفة التقريبية، وكميات المواد، وتوزيع العمالة، والمدة الزمنية المتوقعة، ويمكنك في النهاية استخدام زر **تصدير التقرير** لطباعة التقرير أو حفظه بصيغة PDF من المتصفح.

# علومكم للمقاولات | Al3lomcom Contracting

`kamel3lom`

أداة ويب Static لحساب تقديري أولي لكميات مواد البناء، التكلفة، العمالة، ومدة التنفيذ حسب الدولة ونوع المشروع. تعمل بالكامل من جهة المتصفح بدون Backend أو قاعدة بيانات Server-side، ومهيأة للنشر مباشرة على GitHub Pages.

## تنبيه مهم

هذه النسخة تحتوي على ربط جزئي لمواد السعودية من نشرة `GASTAT APGS` الرسمية، بينما تبقى بقية الدول وبعض البنود غير المتاحة رسميًا موسومة كـ `sample/demo`. لا تعتبر النتائج عرض سعر، أو حصر كميات معتمد، أو بديلًا عن مهندس، مكتب هندسي، مقاول مرخص، أو اشتراطات الجهات الرسمية.

قبل الاعتماد على أي تقرير، يجب استبدال كل بند موسوم `isDemo: true` بأسعار رسمية أو عروض موردين موثقة.

## طريقة التشغيل محليًا

لأن التطبيق يقرأ ملفات JSON عبر `fetch`، يفضل تشغيله من خادم محلي بسيط:

```bash
python -m http.server 5173
```

ثم افتح:

```text
http://localhost:5173
```

يمكن أيضًا استخدام أي خادم Static آخر مثل VS Code Live Server.

## النشر على GitHub Pages

المشروع مهيأ حاليًا للنشر من خلال GitHub Actions عبر الملف:

```text
.github/workflows/deploy.yml
```

وللتأكد من النشر الصحيح:

1. افتح إعدادات المستودع `Settings`.
2. افتح تبويب `Pages`.
3. اجعل مصدر النشر `GitHub Actions`.
4. انتظر اكتمال آخر تشغيل ناجح من تبويب `Actions`.

لا يحتاج المشروع إلى `npm install` أو build.

## ملفات البيانات

- `data/countries.json`: الدول، العملات، المناطق، والضريبة أو الرسوم.
- `data/materials.json`: تعريف المواد ووحداتها وتصنيفاتها.
- `data/prices.json`: أسعار تجريبية sample/demo حسب الدولة والمادة.
- `data/official-price-sources.json`: سجل مصادر رسمية أو مرشحة، مع حالة الربط لكل دولة.
- `data/building-codes.json`: بيانات مرجعية مختصرة لأكواد البناء بدون نسخ النصوص القانونية.
- `data/labor-rates.json`: أجور العمالة اليومية التجريبية.
- `data/calculation-factors.json`: معاملات الكميات، التشطيب، المراحل، الهالك، والاحتياطي.

## مصادر الأسعار الرسمية

تم ربط بعض مواد السعودية من نشرة الهيئة العامة للإحصاء:

- متوسطات أسعار السلع والخدمات `APGS`.
- مواد مستخدمة حاليًا: حديد التسليح، الأسمنت الأسود، الخرسانة الجاهزة، البلوك، الرمل، مخلوط الرمل والبحص، خشب الشدات بتحويل وحدة، وبلاط كسر الرخام.
- المصدر محفوظ داخل `data/official-price-sources.json`.

بالنسبة لقطر، الإمارات، البحرين، عمان، الكويت، مصر، والأردن، أضفت ملف مصادر رسمية مرشحة، لكن لم أفعّل أسعارًا رسمية تلقائية لها إلا بعد توفر رابط رسمي واضح لأسعار وحدات مواد البناء أو ملف دوري قابل للقراءة. هذا مقصود حتى لا ندخل أسعارًا غير موثقة ونقدمها كحقيقية.

## تعديل الأسعار

افتح `data/prices.json` وعدل قيم:

- `priceMin`
- `priceMax`
- `averagePrice`
- `sourceName`
- `sourceUrl`
- `lastUpdated`
- `updateMethod`
- `sourceReliability`
- `isDemo`

عند إدخال أسعار رسمية أو عروض موثقة، غير `sourceReliability` بما يناسب المصدر، واجعل `isDemo` مساوية لـ `false`. أي بند يبقى `isDemo: true` سيظهر للمستخدم كبند تجريبي.

## تعديل أكواد البناء

افتح `data/building-codes.json` وعدل البيانات المرجعية فقط:

- اسم الكود.
- الجهة المصدرة.
- رابط المصدر الرسمي.
- سنة الإصدار.
- نطاق الكود.
- الحالة.
- ملاحظات مختصرة.

لا تضع نصوص الأكواد أو اللوائح كاملة داخل المستودع.

## منطق الحساب

الدوال الأساسية موجودة في `js/calculations.js`. الحساب يعتمد على:

- إجمالي مساحة البناء = مساحة البناء لكل طابق × عدد الطوابق، مع إضافة القبو عند اختياره.
- تكلفة المواد = الكمية × متوسط السعر من `prices.json`.
- تكلفة العمالة = عدد العمال × أيام العمل × الأجر اليومي.
- تكلفة التشطيب = إجمالي مساحة البناء × معامل التشطيب حسب الدولة والمستوى.
- التكلفة الإجمالية = المواد + العمالة + التشطيب + النقل + المعدات + الهالك + الاحتياطي + الضريبة عند تفعيلها.
- مدة التنفيذ = مجموع مراحل البناء مع تعديلات المساحة والطوابق والإضافات والتشطيب.

كل المعاملات قابلة للتعديل من `data/calculation-factors.json`.

## التصدير والحفظ المحلي

- زر `تصدير التقرير` يفتح تقريرًا قابلًا للطباعة عبر `window.print`.
- التقرير يتضمن بصمة `kamel3lom`.
- آخر مشروع يتم حفظه في `LocalStorage` على متصفح المستخدم فقط.
- لا توجد بيانات ترسل إلى خادم خارجي.

## GitHub Actions

يوجد Workflow أولي في:

```text
.github/workflows/update-prices.yml
```

الـ Workflow يشغل `scripts/update_saudi_gastat_prices.py` لتحديث بنود السعودية من PDF رسمي من GASTAT، ثم يعمل commit تلقائيًا إذا تغيرت الأسعار. لا يجلب أسعارًا وهمية ولا يحدث أي دولة لا يوجد لها مصدر رسمي قابل للقراءة.

يمكن تشغيله يدويًا وتمرير رابط PDF رسمي أحدث من صفحة Actions، أو تركه يستخدم الرابط المسجل في `data/official-price-sources.json`.
